1. Rfxcom

1.1. Beschreibung

Plugin permetant d’utiliser une clef rfxcom

1.2. Konfiguration

Le plugin RFXCOM permet de dialoguer avec l’ensemble des périphériques compatibles avec le module RFXCOM USB. Parmi ces périphériques on retrouve les marques: Oregon, LaCross, Chacon, X10, DI-O, Blyss etc…​

Après l’avoir téléchargé sur le Market, il sera nécessaire de configurer le port sur lequel est connecté le RFXCOM USB, du type /dev/ttyUSB. Une liste déroulante propose les ports USB actifs. Et la configuration des protocles actifs : En effet le module RFXCOM peut dialoguer avec énormément de protocole. Mais pour ne pas le sur charger, par défaut il n’y a que certains protocoles qui sont activés. Avec ce menu vous allez pouvoir choisir quel protocole vous voulez activer.

Une fois configuré, on accède à la page du plugin RFXCOM.

A gauche, la liste des modules RFXCOM, et au centre les onglets Général, Information et Commandes.

Le menu à gauche présente l’ensemble des modules RFXCOM détectés et configurés sur son installation domotique. La plupart des modules RFXCOM sont automatiquement reconnus et configurés, il n’y a pas d’inclusion à faire. Quelques rares équipements ont besoin d’être initialisés (voir la documentation de l’équipement).

Le bouton "Ajouter équipement" permet d’ajouter des équipements spécifiques RFXCOM qui nécessitent une synchronisation.

Lorsqu’on passe en mode Expert, on a accès à 1 nouveaux boutons.

Le premier bouton du milieu permet de visualiser tous les dialogues du module RFXCOM.

L’onglet Information précise le type de l’équipement, dans la plupart des cas le type est automatiquement rempli.

L’onglet général permet de choisir le nom de l’équipement, sa destination dans l’arborescence de sa domotique, la catégorie du module, la possibilité de rendre inactif le module dans Jeedom, ou encore rendre visible ou invisible le module dans l’interface.

L’onglet détaillant l’ensemble des commandes (certains éléments ne sont disponibles qu’en mode expert)

Ces commandes sont automatiquement remplies pour la plupart des équipements RFXCOM. Les paramètres utiles sont Historiser, Afficher(la commande), Evènement (permet de forcer la demande d’info sur le module).

Les périphériques type RFXCOM sont reconnus automatiquement par Jeedom, il y a très peu de paramétrage à faire.

1.4. Ajouter une prise

Ce mode opératoire est valable pour les prises, récepteurs luminaire, etc…​ Rendez vous dans le menu "Plugin", puis "Protocole domotique" et enfin "RFXcom".

Cliquez sur "Ajouter" pour ajouter un module, nommez votre module et cliquez sur "D’accord".

Sélectionnez "Prise - Defaut" dans "Equipement", puis cliquez sur "Sauvegarder".

  • Mettre votre module/prise en mode apprentissage. (Cliquez sur le bouton apprentissage de votre module/prise, certaines prises ne possèdent pas de bouton, il suffit de débrancher la prise puis de la rebrancher et pendant les 3 premières secondes suivant le branchement la prise est en mode apprentissage).

  • Cliquez sur "Tester".

  • Si la connexion s’est bien déroulée, vous pouvez cliquer sur "Sauvegarder", dans le cas contraire vous pouvez recommencer cette étape.

Votre module/prise est maintenant connecté avec JEEDOM, pensez à l’activer et le rendre visible pour le voir apparaitre sur le Dashboard.

1.5. Ajouter un store somfy

Depuis la sortie du nouveau module RFXcom RFXtrx433E, il est désormais possible de piloter ses périphériques Somfy RTS (RFY) depuis Jeedom. (Qui prend en charge ce module via le plugin RFXCOM)

Pour ajouter un nouveau périphérique, se rendre dans le menu Plugins > Protocole Domotique > RFXcom

Cliquez sur le bouton +Ajouter et donner un nom à votre périphérique. Ex : « Volet Véranda ».

Se rendre sur le nouveau Périphérique créé, et dans le menu déroulant Equipement, choisir « RTS Store Somfy – Defaut » , en profiter pour lui définir un objet parent, une catégorie si vous le souhaitez et cliquer sur enregistrer.

Les commandes utiles pour gérer le volet roulant sont automatiquement générées. Il ne reste plus qu’à appairer le volet avec ce périphérique Jeedom. Pour cela, appuyez sur le bouton d’appairage de la télécommande du volet que vous souhaitez commander pendant 2 à 3 secondes, jusqu’à ce que le volet fasse un léger aller/retour.

Une fois que le volet a signifié qu’il est en attente de commande pour l’appairage, cliquer sur le bouton Tester correspondant à la commande Programme dans Jeedom.

Pour valider l’apprentissage de la nouvelle commande, le volet va alors faire de nouveau un bref aller/retour.

Afin de tester tout de suite, vous pouvez cliquer sur les boutons Tester correspondants aux commandes « Monter, Descendre, Stop ». Le bouton « My » correspond à la commande Stop.

Activez et Rendez Visible le périphérique avec les cases à cocher correspondantes afin de pouvoir le visualiser et l’utiliser dans le dashboard.

Vous pouvez désormais commander votre volet Somfy RTS avec Jeedom.

1.6. FAQ

Pourquoi l’association d’un prise est plus complexe sur Jeedom que sur d’autre solution domotique ?

Sur les autres solution domotique il suffit souvent juste de cliquer sur les boutons de la télécommande pour pouvoir piloter la prise que la télécommande pilote, pourquoi ce n’est pas le cas sous Jeedom ? Jeedom dissocie la télécommande de la prise, avec la méthode precedente vos boutons sont obligé de piloter la prise et si vous cassez l’association entre la prise et la télécommande la solution domotique ne peut plus piloter la prise. Sous Jeedom c’est different, l’association est plus complexe mais la télécommande n’ai pas obligé d’avoir X boutons monopolisé par la commande de la prise et il peuvent servir a piloter un groupe de prise mixant plusieurs technologie (zwave, enocean et rfxcom par exemple).
