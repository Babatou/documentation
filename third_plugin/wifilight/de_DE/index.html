1. Wifilight

1.1. Präsentation

Dieses Plugin erlaubt es, mit vielen LED-Bändern oder LED-Glühbirnen mit der Funkfrequenz 2.4 GHz, über eine WLANbox zu kommunizieren, das mit dem Produkt verkauft wurde. Das Plugin kommuniziert mit der WLANbox, aber nicht direkt mit den LEDs.

1.2. Kompatibilität und Beschränkungen

1.3. Produits compatibles

Il existe de nombreux produits compatibles avec le plugin mais de marques très différents ou sans marque.

Les produits connus comme compatibles :

  • Ampoules Mi.Light/EasyBulb/LimitlessLED : pas de retour d’état

  • Stgrip-led Mi.Light/EasyBulb/LimitlessLED : pas de retour d’état

  • Contrôleur led Mi.Light/EasyBulb/LimitlessLED V3.0 à V5.0 : pas de retour d’état

  • Contrôleur led Mi.Light/EasyBulb/LimitlessLED V6.0 : pas de retour d’état

  • Contrôleur bandeau led LW12/Lagute : ne gère pas le blanc

  • Contrôleur bandeau led Magic UFO : gère le blanc

  • Ampoules WiFi Xiaomi Yeelight blanches et couleur avec retour d’état

  • Strip-led WiFi Xiaomi Yeelight couleur avec retour d’état

Produits pouvant être compatibles :

  • Ceux compatibles avec le logiciel Magic Home de type 1 et 3

Produits incompatibles et qui ne le seront pas :

  • les ampoules ou contrôleurs de bandeau led qui contiennent un récepteur bluetoooth au lieu d’un recepteur radiofréquence 2.4 Ghz ou WiFi.

  • les contrôleurs de bandeau led ou d’ampoules et les ampoules qui utilisent une connexion WiFi point à point avec l’application mobile.

1.4. Test de compatibilité

  • Sur demande, un plugin de test peut être fourni.

  • Il est conseillé de se renseigner sur le forum pour connaitre la compatibilité d’un produit peu diffusé.

  • Aller sur le forum de Jeedom ici : https://forum.jeedom.fr/viewtopic.php?f=28&t=2840

1.5. Limitations

Mi.Light/EasyBulb/LimitlessLED :

  • toutes les fonctionnalités sont prises en compte par le plugin.

LW12/Lagute :

  • la programmation des modes custom n’est pas possible, il faut utiliser l’application fournie avec le contrôleur (Magic Home). Par contre, les modes customs peuvent être déclenchés avec le plugin. Le retour d’état de l’intensité n’est pas géré par le plugin car n’est pas natif au contrôleur.

Magic UFO :

  • la programmation des modes custom, le mode radio et les timers ne sont pas gérés. Il faut alors utiliser l’application fournie avec le contrôleur (Magic Home). Par contre, les modes customs peuvent être déclenchés avec le plugin. Le retour d’état de l’intensité n’est pas géré par le plugin car n’est pas natif au contrôleur.

Xiaomi Yeelight :

  • les commandes HSV ne sont pas gérées.Flow et scène le sont en créant des commandes avec le code JSON correspondant à l’effet recherché (voir la documentation de l’API YeeLight).

1.6. WiFi Modul Konfiguration

1.6.1. Installer les leds

Attention, le plugin est uniquement compatible avec les leds wifi des marques Mi.Light/EasyBulb/LimitlessLED et les ponts (bridge) V3 à V6, le contrôleur de bandeau à leds LW-12, les contrôleurs compatibles Magic UFO ainsi que Les ampoules Xiaomi YeeLight

Utiliser la documentation du constructeur et vérifier le fonctionnement avec la télécommande radiofréquence du constructeur. Noter le canal (1 à 4) (inutile pour le contrôleur LW-12 ou les contrôleurs compatibles Magic UFO et les ampoules Xiaomi YeeLight).

Télécharger l’application mobile du constructeur et suivre les indications afin de contrôler les leds avec le mobile.

Consultez l’aide et les forums du constructeur des lampes.

1.6.2. Configurer le routeur

Si vous utilisez une box domotique aller dans la partie configuration DHCP et modifier l’attribution de l’adresse IP du module wifi ou de l’ampoule pour qu’elle soit statique. Noter cette adresse. En général elle sera de la forme :

192.168.1.xxx où xxx correspond à l’adresse du module wifi (2 à 254)

Consultez les forums de votre box pour apprendre à configurer votre DHCP.

Après cette modification, vérifier que l’application mobile contrôle toujours les lampes.

Vous pouvez alors passer à la configuration du plugin Wifilight.

1.7. Plugin Installation und Konfiguration

1.7.1. Einrichtung

Pour paramétrer un équipement, choisir le menu Plugins / Objets communicants / Wifilight

Puis cliquer sur le bouton en haut à gauche Ajouter un module Wifi

  • Saisir le nom du module wifi

  • Saisir l’objet parent

  • Choisir la catégorie Lumière

  • Aktivieren und sichtbar zu machen

  • Saisir l’adresse IP du module ou de l’ampoule WiFi

  • Saisir la canal utilisé (ce paramètre est ignoré pour les contrôleurs LW-12 ou compatibles Magic UFO,le pont Mi.Light V6 et les ampoules Xiaomi YeeLight)

  • Saisir le type de lampe (Mi.Light RGBW, Mi.Light RGBW pont V6, Mi.Light blanche, Mi.Light Blanche Pont V6, Mi.Light RGBW WW/CW, Pont Mi.Light V6, LW-12, Magic UFO, Xiaomi Yeelight Blanche, Xiaomi Yeelight RGBW, Xiaomi Yeelight bandeau led). C’est indispensable pour créer l’ensemble des commandes pour le type de lampe ou de pont utilisé

  • Saisir le nombre d’envois de commandes : permet de répéter la commande pour un équipement éloigné en cas de mauvaise transmission

  • saisir le délai d’envoi en cas de répétition

1.7.2. Ajout des commandes

A la sauvegarde du module, les commandes sont automatiquement créées.

Le nom des commandes peut être modifié. Les commandes automatiquement créées et effacées sont recréées lors d’une sauvegarde. Toutes les commandes sont créées et peuvent alourdir l’interface, vous pouvez choisir de ne pas les afficher.

Pour les autres canaux du même module WiFi, créer un nouveau module WiFi de la même façon et modifier le canal.

Pour les Mi.Light/EasyBulb/LimitlessLED, utiliser un équipement avec le canal 0 pour contrôler les 4 canaux en même temps.

1.8. Utilisation du widget

Le widget dans le tableau de bord va permettre de contrôler les leds :

Les widgets sont les widgets standards de Jeedom. Pour le widget color, il est possible de télécharger : dashboard.action.color.Color_default_no_off qui correspond au widget standard mais sans le bouton off.

Attention : il n’y a pas de retour d’état depuis les Mi.Light. Si l’éclairage a été modifié par une télécommande radiofréquence, le plugin ne retournera pas l’information sur le tableau de bord. Pour les LW12/Magic UFO le plugin permet d’obtenir l’état toutes les minutes. La récupération par scénario est possible. L’intensité modifiée avec une app sur mobile sans passer par jeedom ne sera pas répercutée car l’intensité n’est pas gérée par ces contrôleurs. Pour les Xiaomi YeeLight, le plugin permet le retour d’état toutes les minutes de la couleur, l’intensité, ON/OFF, la température de couleur.

1.10. Funktionsweise der Stausrückgabe

1.10.1. Compatibilité du retour d’état

Le retour d’état fonctionne avec les contrôleurs LW12/Lagute, Magic UFO ainsi que les ampoules et bandeau led Xiaomi YeeLight (dans une version future)

1.10.2. Prinzip

Le retour d’état consiste à ce que Jeedom récupère l’état du contrôleur si son état a été changé par un autre maitre que Jeedom : appli portable ou télécommande

1.10.3. Mise à jour périodique de Jeedom

  • LW12/Lagute, Magic UFO, Xiaomi Yeelight : toutes les minutes Jeedom interroge le contrôleur pour connaitre son état et mettre à jour l’allure des widgets du plugin ( sliders et couleur ). Des informations correspondant à l’état sont mises à jour et peuvent être interrogées par les scénarios.

1.10.4. Szenario Update

La commande GetState permet de mettre à jour dans un scénario jeedom avec l’état du contrôleur

1.10.5. Récupération des informations de retour d’état

Toutes les commandes "Info" du widget et qui contiennent Get ou Etat dans leur nom correspondent à un des états du contrôleur : couleur, vitesse, ON/OFF …​

Contrôleurs de type Magic UFO :

  • ColorGet : récupère la couleur au format #RGB (#FF00FF)

  • ModeGet : récupère le mode guirlande

  • SpeedGet : récupère la vitesse du mode guirlande

  • Etat : récupère si le contrôleur est arrêté ou en marche → marche: 1, arrêt : 0

  • SwProgRunGet : récupère si le contrôleur, en mode guirlande, fonctionne ou est en pause → pause : 0, marche : 1

  • WhiteGet : récupère l’intensité du blanc 0 à 255

Contrôleurs de type LW12/Lagute :

  • ColorGet : récupère la couleur #RGB (#FF00FF)

  • ProgGet : récupère le n° du mode guirlande

  • SpeedGet : récupère la vitesse du mode guirlande

  • Etat : récupère si le contrôleur est arrêté ou en marche → marche: 1, arrêt : 0

  • SwProgRunGet : récupère si le contrôleur, en mode guirlande, fonctionne ou est en pause → pause : 0, marche : 1

Ampoules et strip-led Xiaomi Yeelight :

  • ColorGet : récupère la couleur #RGB (#FF00FF) (pour les lampes compatibles)

  • Etat : récupère si le contrôleur est arrêté ou en marche → marche: 1, arrêt : 0

  • IntensityGet : résupère l’intensité

  • KelvinGet : résupéère la température de couleur (pour les lampes compatibles)

1.11. Mode scène Xiaomi Yeelight

1.11.1. Konfiguration der Lampe

Il est indispensable d’activer le mode développeur via l’application Xiaomi Yeelight.

1.11.2. Mode Scène Xiaomi Yeelight

Il est possible de configurer les modes scène. Plusieurs modes scènes sont préprogrammés mais il est possible d’ajouter d’autres modes scène.

Il suffit de respecter certaines conditions :

  • Ajouter une commande wifilight action de type Défaut

  • Son nom interne doit commencer par Scene. Exemples : SceneSoleil - Scene025

  • Dans parametres, mettre la commande scène Yeelight, par exemple : "id":1, "method":"set_scene","params":["cf",0,0,"500,1,255,100,1000,1,16776960,70"]

  • Ne pas mettre les accolades de début et de fin ainsi que les caractères de retour à la ligne, le plugin les ajoutera automatiquement

  • S’inspirer des commandes préconfigurées pour créer ces modes scène supplémentaires.

1.12. FAQ

Quelles ampoules peuvent être utilisées ?

Les produits connus comme compatibles :

  • Ampoules Mi.Light/EasyBulb/LimitlessLED sur les ponts V3 à V5 : pas de retour d’état

  • Contrôleur led Mi.Light/EasyBulb/LimitlessLED V3 à V5: pas de retour d’état

  • Contrôleur led Mi.Light/EasyBulb/LimitlessLED V6.0

  • Contrôleur led LW12/Lagute : ne gèrent pas le blanc

  • Contrôleur led Magic UFO : gère le blanc

  • Ampoules Xiaomi Yeelight blanches et couleur

  • Contrôleur de led Xiaomi Yeelight couleur

Produits pouvant être compatibles :

  • Ceux compatibles avec le logiciel Magic Home de type 1 et 3

Produits incompatibles et qui ne le seront pas :

  • les ampoules ou contrôleurs de bandeau led qui contiennent un récepteur bluetoooth au lieu d’un recepteur radiofréquence 2.4 Ghz ou WiFi.

  • les contrôleurs de bandeau led ou d’ampoules qui utilisent une connexion wifi point à point avec l’application mobile.

Les ampoules et contrôleurs d’autres marques ou les clones ne sont pas supportés ou doivent être testés au préalable.

Un plugin de test pourra être fourni pour vérifier la compatibilité des leds, contacter l’auteur via le forum.

Rien ne se passe

Faire d’abord fonctionner les lampes avec l’application mobile fournie par le constructeur.

Utiliser le bouton tester dans le menu Plugin/Objets Connectés/Wifilight.

Aucune aide ne sera apportée sans que les lampes soient opérationnelles avec la télécommande ou l’application du téléphone portable.

Je ne sais pas configurer ma box internet

Aucune aide ne sera apportée sur la box et les notions nécessaires pour configurer le routeur pour attribuer une adresse IP fixe. Consulter les forums de la box.

La gestion de l’intensité des ampoules blanches Mi.Light/EasyBulb/LimitlessLED n’est pas pratique

Le constructeur des leds n’a pas prévu de pouvoir affecter directement l’intensité de l’ampoule. On ne peut qu’incrémenter ou décrémenter par rapport à la valeur précédente. Le plugin ne fait que reproduire ce fonctionnement. Le curseur qui est proposé est en conséquence capricieux.

Y-a-t-il un retour d’état ?

Les ampoules RGBW Mi.Light/EasyBulb/LimitlessLED ne possèdent pas de retour d’état. Cela signifie que si, par exemple, l’ampoule est éteinte avec la télécommande ou avec l’application du téléphone portable, le plugin n’est pas prévenu. Ainsi Jeedom pourra afficher que la lampe est allumée alors qu’elle a été éteinte par ailleurs. L’interface utilisateur de Jeedom ne correspondra donc pas à la réalité. Un moyen de contourner partiellement ce problème est d’utiliser des télécommandes connectées à Jeedom, il sera alors possible d’indiquer au widget l’action réalisée.

Le contrôleur LW-12 et le contrôleur Magic UFO peuvent être interrogés pour connaitre leur état et sont interrogés toutes les minutes, seule l’intensité n’est pas répercutée si elle a été modifiée autrement qu’avec Jeedom.

Les ampoules et bandeaux de led Xiaomi Yeelight possèdent un retour d’état mis à jour toutes les minutes, Jeedom peut interroger leur état dans un scénario.

Impossible de faire fonctionner les ampoules Xiaomi Yeelight

Il est indispensable d’activer le mode développeur via l’application Xiaomi Yeelight.

Comment obtenir de l’aide ?

Aller sur le forum de Jeedom ici : https://forum.jeedom.fr/viewtopic.php?f=28&t=2840

1.13. Probleme

Erreur lors de l’envoi de commande / roue crantée sans arrêt / Emission sans arrêt

  • les équipements wifilight doivent être mis à jour

  • → aller dans chaque équipement et sauvegarder 2 fois

  • → tester avec un nouvel équipement

Pont Mi.Light V6 : commande non prise en compte

  • les ordres sont envoyés trop rapidement

  • dans les scénarios, mettre des pauses du durée suffisante

Erreur wifilight.class.php on line xxxx

  • vérifier l’existence de la commande "SwOnOffGetxxx".

  • si elle n’existe pas : resauvegarder 2 fois votre équipement

  • si elle existe : supprimer la commande et resauvegarder 2 fois

  • → la commande sera recéée avec la bonne configuration

LW12 et Magic UFO

  • Les nouvelles fonctionnalités sont expérimentales (retour d’état)
